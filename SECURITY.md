## Reporting a Vulnerability

If you discover a security vulnerability within AEGIS itself (not the attack techniques it tests), please follow responsible disclosure:

### Do NOT

- ‚ùå Open a public GitHub issue
- ‚ùå Disclose publicly before we've had a chance to fix it
- ‚ùå Use the vulnerability for malicious purposes

### Do

1. **Email**: Send details to [security@yourdomain.com] (replace with your email)
2. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if you have one)
3. **Wait**: We'll respond within 48 hours
4. **Coordinate**: Work with us on disclosure timeline

## Scope

### In Scope
- Vulnerabilities in AEGIS code itself
- API security issues
- Authentication/authorization bypasses
- Injection vulnerabilities in the tool

### Out of Scope
- The attack techniques AEGIS uses (these are intentionally included)
- Vulnerabilities in target LLMs (report to their vendors)
- Social engineering attacks
- Physical security
